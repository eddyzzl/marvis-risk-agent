# MARVIS 领域词汇表

本文件只是词汇表：记录项目的统一语言（ubiquitous language），不含实现细节。实现决策见 `docs/plans/`（计划与 spec）与 `docs/reviews/`（审查报告）。

## 质量与信任

- **信任层（Trust Layer）**：让平台呈现的每个决策数字"可以直接进决策"的全部机制的统称——语义契约门、双路对账、数字血缘、三层数据判据。区别于"能力层"。
- **能力层（Capability Layer）**：扩展平台覆盖信贷风控生命周期阶段的功能（标签构造、部署、监控自动化、外部数据接入等）。
- **数据契约（Data Contract）**：某个工具对输入数据未经校验即信任的语义/形状假设（如"标签列 1=坏"、"每个坏客户只在变坏那期标 1"）。契约破裂时静默出错是平台最坏的失败模式。
- **语义契约门（Semantics Gate）**：把一条关键数据契约从"隐含假设"升级为"强制用户确认"的 typed gate。先例：NaN 标签确认门。
- **脏形状（Dirty Shape）**：真实业务数据中会击穿某条数据契约的具体形态（sentinel -999、快照型面板、NULL 标签、float64 存储的证件号、空白键、零填充键等）。
- **三层数据判据（Three-Tier Data Criteria）**：Phase 收口的主出口判据，取代"审查无新发现"：①对抗注入回归（脏形状生成器，进 CI）②公开数据集端到端 + KS 基准 ③真实业务材料人工对账。
- **对账（Reconciliation / 双路对账）**：同一 headline 数字由两条独立实现（如 DuckDB SQL 路径与 pandas 路径）各算一遍并强制比对，不一致即阻断红旗。"复式记账"思想在指标上的应用。
- **轻血缘（Light Provenance）**：gate 数字携带的最小溯源元组（数据集指纹、代码版本、参数、seed），区别于完整血缘图与一键重放（均为后置项）。
- **范围化 review（Scoped Review）**：仅审当期 diff 的一轮审查，只修 critical/high，不循环。区别于全量 review（大重构后按需触发）。

## 标签与口径

- **标签语义（label_semantics）**：面板数据中坏标签列的两种互斥语义——**增量型（incremental）**：客户仅在变坏的那一期标 1；**快照型（snapshot / ever-bad）**：客户从变坏起每期都标 1。vintage 累计口径只对增量型正确；快照型输入必须走边际口径或先转换。
- **定坏口径（Bad Definition）**：把逾期表现（DPD/状态串）映射为 0/1 坏标签的规则，含观察期、表现期、逾期阈值（30+/60+/90+）三要素。
- **成熟度（Maturity）**：一个放款 cohort 的表现期是否已闭合到足以定坏。不成熟 cohort 的标签不可用于建模。
- **口径变化（Metric Basis Change）**：bug 修复导致同一数据上历史报告与新代码数字不一致。平台约定：直接修正、不做旧值兼容、在发布说明中列明。

## 既有核心词（沿用）

- **确定性内核（INV-1）**：同输入必同输出的指标计算层；两处可分歧的实现是它的直接违反。
- **锚定左连接**：以样本表为锚、行数守恒的 join 范式，行数偏差即硬失败回滚。
- **typed gate / 强制确认**：不可绕过的人工确认点，携带结构化证据 payload。
- **evidence envelope**：工具输出随附的结构化证据，gate 渲染的数据来源。
