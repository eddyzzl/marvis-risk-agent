# 真实材料对账 Checklist（三层数据判据 · 第三层）

> **归属**：[v2-trust-first-plan.md](v2-trust-first-plan.md) T4-3；三层数据判据的**人工验收层**（第一层=对抗形状注入器进 CI、第二层=公开数据集端到端+KS 基线、第三层=本清单）。
> **用途**：Phase T 收口的人工步骤。不进 CI——它需要真实业务材料和外部口径（财务/拨备/已知报表），只有人能签字。**每接入一份新真实数据集，跑一遍本清单**。
> **依据**：2026-07-04 全量读码报告确认的 8 条数据语义 bug 全部偏向"真实数据比测试数据脏"——本清单逐项对外部事实核对平台产出的 headline 数字，把"数字能直接进决策"从假设变成签字。

## 使用方法

对每一份接入的真实业务数据集，逐项执行、记录实测值与外部口径值、签字。任一项对不上 → 阻断收口，回到对应修复项排查。

---

## A. 数据契约与脏形状（对应 T1，接入即查）

| # | 检查项 | 怎么验 | 通过判据 | 实测 | 签字 |
|---|---|---|---|---|---|
| A1 | sentinel 值 | 数值列是否含 -999/-9999/9999 等哨兵 | 若有，确认预处理链含 sentinel step（评分时 -999 不当真值） | | |
| A2 | vintage 标签语义 | bad 列是"当期新坏"还是"快照 ever-bad" | 快照型必须在 gate 声明 snapshot，曲线值合理（非逐期虚高） | | |
| A3 | NULL/无标签行 | 样本是否含空/非 0/1 标签 | bad_rate 分母剔除未标注行，gate 显示 unlabeled_count | | |
| A4 | 长 ID 键 | 证件号/手机号是否 float64 存储、>15 位 | join 键匹配正常或触发精度红旗（非 100% 静默不匹配） | | |
| A5 | 空白/零填充键 | join 键是否含空白或前导零编码 | 空白键不错配（特征列 NULL）、零填充键跨文件 dtype 一致 | | |
| A6 | 时间 OOT | 是否指定了 time_col | 非别名列名也触发时间 OOT（非静默随机切分） | | |

## B. Headline 数字对外部口径（核心对账，双路对账之外的人工层）

| # | 数字 | 平台产出 | 外部口径来源 | 通过判据 | 实测 vs 口径 | 签字 |
|---|---|---|---|---|---|---|
| B1 | **vintage 累计坏账率** | 策略 vintage 曲线 | 财务/风险部门的账龄坏账口径 | 同账龄同 cohort 量级一致（非差一个数量级） | | |
| B2 | **组合 EL（total_el）** | analysis EL 估计 | 拨备/减值口径（IFRS9 或内部） | 参考快照月 EL 与拨备口径量级一致（非 ~N 倍虚高） | | |
| B3 | **分群 bad_rate** | slice_aggregate | 已知的分渠道/分月坏账报表 | 各切片坏率与报表一致（含无标签行剔除后） | | |
| B4 | **模型 KS** | 建模报告 test/OOT KS | 独立复算或历史同类模型 | OOT KS 在合理区间，与冠军 gate 证据显示的 selection_metric 自洽 | | |
| B5 | **join 匹配率** | join propose/confirm gate | 手工核对样本键匹配 | 双路对账无红旗；抽样键匹配与显示一致 | | |

| # | 检查项 | 怎么验 | 通过判据 | 签字 |
